About these settings

Storage: all timestamps are stored in UTC in the database, for consistency regardless of server timezone.

Display conversion: timestamps are converted from UTC to your selected timezone in the UI, API responses and charts.

Changing timezone: takes effect immediately for existing and new timestamps — no data migration needed.

Language: only English is supported today; more will follow.

Data retention: a sliding window — only the last N days of collected data are kept; older data is pruned automatically each day and cannot be recovered.
